画像の最適化により『PageSpeed Insights』の点数を上げてウェブサイトを高速化する方法

ウェブサイトにおいて，高速化を目指すためには画像の最適化が必要不可欠です。画像が最適化されていないと低速回線を使用しているユーザーにとって非常にロード時間が長くなり快適性が損なわれてしまいます。このサイトでは画像自体の最適化とサイト上の画像表示の工夫を紹介します。

1.『PageSpeed Insights』で自分のウェブサイトを計測

まず，『PageSpeed Insights』とは、ウェブページのコンテンツを解析し、ページの読み込み時間を短くするための方法を提案してくれる便利なものです。それでは，『PageSpeed Insights』で自分のウェブサイトを計測しましょう。

このように計測結果が表示されます。注目してほしいのが下の『改善できる項目』にある欄です。ウェブサイトに画像を使用していて最適化してないと，画像関連の項目が多数あると思います。この項目に従って画像の最適化等の方法をお教えしていきます。

2.画像自体の最適化方法

●画像最適化する必要性

まず，なぜ画像を最適化する必要があるのかというと，最適化してないと画像の容量が大きすぎてウェブサイトの表示時間が遅くなるからです。5Gや有線LAN接続のパソコンのような高速な回線であれば極端に大きい容量の画像等でなければ快適に表示されるでしょう。しかし，低速回線を使用している人もいるようにみんながそうではないのでやはり最適化している方が良いに越したことはないでしょう。

●画像最適化する必要性

次に，画像のサイズについて，おすすめの解像度の横幅がこちらです。

サムネイル画像 画像
パソコン用
pc(960px~)		 1200px 795px
タブレット用
tablet(481px~959px)		 959px 795px
スマートフォン用
mobile(~480px)		 480px 480px
記事一覧用サムネイル画像 200px

『画像』とはサムネ以外にウェブサイトに使用する画像のことです。

次に，画像のdpi(dots per inch)は72dpiがウェブサイトでは定番です。

●『GIMP』を使用した画像最適化性

以上のように画像を最適化するのに使用するのは『GIMP』という無料ソフトです。他の画像ソフトでもおそらくできると思うのでそちらでも構いません。導入方法は省略します。今回は『GIMP』で『サムネ以外にウェブサイトに使用する画像』最適化の方法を紹介します。

まず，画像を入れます。次に上部にある『画像』の『画像の拡大・縮小』をクリック。

すると，画像の解像度やdpiを変更する欄が出てきます。先程のおすすめの解像度表を参考に値を入力します。なお，値の単位は解像度は『px』，dpiは『ピクセル/in』です。

『拡大・縮小』をクリックすると変更されます。

『ファイル』の『名前を付けてエクスポート』をクリック。画像名を入れ，語尾に

『画像名-pctablet.webp』

をつけます。pcとtablet用の画像なのでpctabletとしていますが，好きなように付けても構いません。

次に，圧縮率や個人情報等削除などをいじります。私はwebp，jpegともに圧縮率は50%，個人情報等は全部削除しています。

『エクスポート』をクリックして画像を生成します。

『webp』非対応ブラウザ用に『jpeg画像』も生成します。

『画像名-pctablet.jpeg』

同様に，スマートフォン向けの画像もつくります。画像サイズを幅480pxにし，72dpiにします。画像名は

『画像名-mobile.webp』

『画像名-mobile.jpeg』

の2種類の画像をつくります。

同じ画像に対して合計で4種類の画像をつくりました。

  • 『画像名-pctablet.webp』
  • 『画像名-pctablet.jpeg』
  • 『画像名-mobile.webp』
  • 『画像名-mobile.jpeg』

なお，サムネイル画像の場合は同様に8種類の画像をつくります。

●HTMLの書き方

前項でパソコン用，(タブレット用，)スマートフォン用と合計4種類(サムネイル画像は6種類)の画像をつくりました。それらを画像を入れたいところに以下のようなコードを入力します。

<p class="img">
    <picture>
        <source type="image/webp" media="(min-width: 481px)" data-srcset="画像名-pctablet.webp">
        <source type="image/webp" media="(max-width: 480px)" data-srcset="画像名-mobile.webp">
        <source type="image/jpeg" media="(min-width: 481px)" data-srcset="画像名-pctablet.jpeg">
        <source type="image/jpeg" media="(max-width: 480px)" data-srcset="画像名-mobile.jpeg">
        <img class="lazyestload" src="../img/icon_loader_f_ww_01_s1.gif" data-src="画像名-pctablet.jpeg" alt="画像名">
    </picture>
</p>

まず，1行目と最後の行の<p>タグで全体を覆っているのは画像が文章の途中に入ったり複数の画像が隣り合うのを防ぐためです。また『class="img"』は画像が中央に来るようにCSSでしています。そのコードも一応載せておきますのでCSSに書いてください。

.img {
    text-align: center;
}

2行目と最後から2番目の行は『<picture>』で覆います。

3行目でパソコンとタブレット用画像をwebp対応ブラウザに対して表示します。『type="image/webp"』はwebp対応ブラウザかどうかの条件です。対応していれば適用されます。『media="(min-width: 481px)"』は画面幅が最低481px以上の場合適用されます。つまり，タブレットとパソコンのように481px以上の画面幅の場合適用されるということです。『data-srcset="画像名-pctablet.webp"』で表示させたいpctablet用のwebp画像名を入力します。

4行目は『type="image/webp"』でwebp対応ブラウザかつ『media="(max-width: 480px)"』で最大480px以下つまりスマートフォンのように480px以下の端末に対して適用します。『data-srcset="画像名-mobile.webp"』で表示させたいmobile用のwebp画像名を入力します。

5行目は『type="image/jpeg"』でjpeg対応ブラウザに対して適用します。webp非対応ブラウザ用のためです。『media="(min-width: 481px)"』で481px以上の画面幅に対して適用します。『data-srcset="画像名-pctablet.jpeg"』で表示させたいpctablet用のjpeg画像名を入力します。

5行目は『type="image/jpeg"』でjpeg対応ブラウザに対して適用し，『media="(max-width: 480px)"』で480px以下の端末に対して適用します。『data-srcset="画像名-mobile.jpeg"』で表示させたいjpegのmobileの画像名を入力します。

6行目は3～5行目に当てはまらなかったものに対して表示させます。『class="lazyestload"』は次の項で説明しますが，『レイジーロード』というのを使用するために必要な記述です。『src="../img/icon_loader_f_ww_01_s1.gif"』は『data-src』で指定された画像を表示する前に表示する画像等です。私はgifを使用しています。こちらも『レイジーロード』を使用するのに必要な記述です。『../img/icon_loader_f_ww_01_s1.gif』はファイルの場所や使用する画像等で変わりますのでもし違うものを使用する場合は変更してください。『data-src="画像名-pctablet.jpeg"』で表示させたいpctablet用のjpeg画像名を入力します。『alt="画像名"』はいずれの画像も表示できない場合(画像自体がないなど)に指定した文字が表示されます。

3.ウェブサイト上の画像表示の工夫

●『レイジーロード』の導入方法

まず，レイジーロードとは画像を遅延読み込みする手法です。ウェブサイトを開いたときには画像が表示されておらず，画面が画像の位置に到達すると表示されます。メリットとしては転送量の節約，ページの表示速度向上に繋がります。

次に，フッターなどの『</body>直前』に以下のコードをコピペしてください。すべてのページです。

<script>!function () { var t, e; t = function () { !function () { for (var t = document.querySelectorAll("img.lazyestload"), e = t.length; e--;) { var r = window.innerHeight, n = t[e].getBoundingClientRect(), a = n.top - r, i = n.bottom; if (a <= 100 && -100 <= i) { if (t[e].getAttribute("data-src") && (t[e].src = t[e].getAttribute("data-src")), t[e].getAttribute("data-srcset") && (t[e].srcset = t[e].getAttribute("data-srcset")), "PICTURE" === t[e].parentElement.tagName) for (var o = t[e].parentElement.querySelectorAll("source"), s = o.length; s--;)o[s].srcset = o[s].getAttribute("data-srcset"); t[e].addEventListener("load", function () { this.classList.remove("lazyestload") }) } } }() }, window.addEventListener("scroll", function () { clearTimeout(e), e = setTimeout(t, 200) }), t() }();</script>

次に，前項の『data-src』を表示する前の画像等を用意します。私はGIFを使用しています。

ダウンロードは『こちら』です。サーバーのimgフォルダに入れましょう。

もし他の画像等を使用する場合は前項の『src="../img/icon_loader_f_ww_01_s1.gif"』を変更しましょう。

これで『レイジーロード』が使えるようになってるはずです。試しに，ブラウザで開き画像が

➩

となれば『レイジーロード』が適用されています。

ウェブサイトがどのくらい速くなったか『PageSpeed Insights』で自分のウェブサイトを再び計測しましょう。点数が上がっていることでしょう。
